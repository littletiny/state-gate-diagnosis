1. **State-Gate 模型**
   - State: 离散状态节点（变量、标志位、计数器）
   - Gate: 基于 State 做决策的关键代码位置
   - Map: State 与 Gate 之间的拓扑关系

2. **ECTM 证据链方法**
   - 观察: 定位关键代码
   - 假设: 提出行为理论
   - 验证: 追踪调用链和边界条件
   - 证据: 结论尽量关联到源码位置或函数名

3. **无阶段约束**
   - 不需要先 Discovery 再 Analysis 的固定顺序
   - 你可以随时创建 State、Gate 或 Map，只要基于源码证据
